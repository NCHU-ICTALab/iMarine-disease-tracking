# 資料來源與真假對照

> 一句話總結：**疫情資料是真的（2026 年 6 月底的官方最新資料），船是假的（測試用的 6 艘模擬船）。**
> 系統做的事，就是拿「模擬船」去比對「真實疫情」，示範如果船是真的、警報會長什麼樣。
> 疫情這一半已經是真的，只要把模擬船換成真實 AIS，同一套引擎即可對真船發真警報。

## 一、真資料（從官方網站即時抓取）

### 1. 台灣疾管署 — 國際旅遊疫情建議等級表 ✅ 真
- **內容**：台灣疾病管制署每日公布「哪個國家正在流行什麼傳染病、警戒到第幾級」。
- **為何可信**：政府官方公告，是本系統判斷風險的**主要依據**。
- **資料時間**：每日更新；本次抓取最新一筆為 **2026-06-30**（例：烏干達馬堡病毒、寮國小兒麻痺、巴基斯坦瘧疾），共約 2,400 多筆，入庫 2,018 筆。
- **來源網站**：疾管署開放資料平台 https://data.cdc.gov.tw/
- **實際下載端點**：
  `https://www.cdc.gov.tw/CountryEpidLevel/ExportCSV?type=0&fileName=TCDCTravelAlertAll.csv`
- **對應程式**：`app/sources/cdc_scraper.py`
- **實測欄位**：`effective`(生效日)、`alert_disease`(疾病)、`severity_level`(警戒等級)、`ISO3166`(國別碼)、`areaDesc`(地區)、`circle`(經緯度)、`web`(來源連結)。

### 2. 世界衛生組織 WHO — 疾病爆發新聞（Disease Outbreak News, DON）✅ 真
- **內容**：WHO 公布的全球重大疫情事件；用來補強疾管署資料，並標記「國際關注公共衛生緊急事件（PHEIC）」。
- **資料時間**：即時；本次抓取最新一筆為 **2026-06-25**（印度立百病毒 Nipah），另有 6/24 黃熱病等。
- **來源網站**：https://www.who.int/emergencies/disease-outbreak-news
- **實際 API 端點**（OData JSON）：
  `https://www.who.int/api/news/diseaseoutbreaknews`
- **對應程式**：`app/sources/who_scraper.py`
- **實測欄位**：`DonId`、`PublicationDate`(公布日)、`Title`(格式「疾病 - 國家」，如 `Nipah virus disease - India`)。

### 4. 病原（疾病）參數表 🟡 醫學知識為真、評分權重為自訂
- **內容**：每種病的潛伏期、傳染途徑、嚴重度。例：COVID 潛伏期 2–14 天、走呼吸道。
- **真假**：潛伏期與傳染途徑等**醫學事實為真**；但「危險程度打幾分」（如傳播力 0.95）是**自訂的評分參數**，供系統計分用，非官方數字，未來可請防疫專家校準。
- **檔案**：`data/pathogens.yaml`；**對應程式**：`app/analysis/pathogens.py`
- **知識依據**：WHO 各疾病說明頁 https://www.who.int/health-topics 、
  WHO《國際衛生條例》IHR https://www.who.int/health-topics/international-health-regulations 、
  疾管署傳染病介紹 https://www.cdc.gov.tw/

## 三、假資料（測試用，之後可替換為真實來源）

### 5. 船舶航跡（6 艘船）❌ 全部為模擬
- **關鍵說明**：系統中的 6 艘船（EVERGLORY01、PACIFICSTAR、OCEANDUKE07、NIPPONMARU9、MANILAEXP12、OLDCALLTEST）——船名、船號、停靠港、抵達日期**全為自建**。
- **為何用假的**：報告書原規劃使用 AIS 即時船舶定位，但真實 AIS 需**付費 API**；MVP 階段先以模擬船把整套邏輯跑通。模擬船的「上一站」刻意設為有真實疫情的國家，以示範比對效果。
- **檔案**：`data/mock_ais.json`；**對應程式**：`app/sources/ais_provider.py`（介面已抽象化）。
- **可替換來源（未來接真實 AIS）**：
  - aisstream.io（免費、需註冊 API Key）：https://aisstream.io/
  - AISHub（需貢獻自有接收器）：https://www.aishub.net/api
  - MarineTraffic / VesselFinder（付費、覆蓋率最佳）

### 6. 細胞簡訊推播 ❌ 模擬（不會真的發簡訊）
- 系統顯示「已推播警報」，但**未真正發送簡訊**。細胞廣播（CBS）屬電信商層級功能，正式落地需與電信商 / NCC 對接；MVP 僅寫入 `notifications` 紀錄並輸出 log。
- **對應程式**：`app/service/notifier.py`（介面已抽象化，未來可換真實發送）。

## 四、「高風險」結果代表什麼？

範例（船為模擬、疫情為真實）：

| 船（模擬） | 上一站（模擬） | 比對到的疫情（真實） | 風險 |
|---|---|---|---|
| OCEANDUKE07 | 印度 → 新加坡 | 印度 Nipah 病毒（WHO 2026/6/25 通報） | 高 |
| MANILAEXP12 | 菲律賓 → 香港 | 香港流感（疾管署通報） | 高 |

意義：**假設真有一艘船這樣航行，系統會拿真實疫情資料算出此警報。** 疫情端已是真實資料，
因此只要把模擬船換成真實 AIS，系統即可對真船發出真警報，**分析引擎無需改動**。
